Devlop

一些适合自己的开发规范，自己规定和重其他优秀代码中吸取后自己使用。

注释

1.一般在功能函数中加注释，一些重要参数和操作可加注释，这样简洁，都写则乱，用英文

    1. 阅读代码的时候，可以多些

    2. /**/这个一般是块注释，//行注释

2.修改代码，要加上自己的注释

//add by jcy for XXX
//end
  1. 每个程序包应该有一个包注释

名字

  1. 总则：开头大写对外可见，小写的内部使用

  2. 程序包一般使用小写，使用简短的单词，建议不要使用._这些，显示的调用包名能够显式的简洁调用而不重复，便于理解

3.go自身不支持get和set，但是通常是需要这些方法的，所以在使用的时候，一般大写代表Get，并不需要在结构体前加一个get，set是可以加一个set的

owner := obj.Owner()
if owner != user {
    obj.SetOwner(user)
}
  1. 接口I开头

单个接口名一般都是使用方法名后加上er来命名，比如Reader，Writer

5.函数变量名尽量使用大小写混合的小驼峰，减少使用下划线

6，结构体的成员都要大写，不然就算同一个文件都不能使用，结构体名在同一个文件内可以小写，但是不同文件使用的时候要大写，才能被引用

6.正常变量

以后命名直接使用名字，不需要带类型，起不了什么作用，显得冗余，这样简洁一点，现在大部分代码都是这样。不能局限于以前的c++风格

/废弃/ int n开头

string str开头 []string sstr []byte s开头

bool b开头

struct 使用struct或者struct缩写开头

map m开头

interface。I开头

channel。 c开头

全局变量 G开头,类型还是使用上面的那种类型

指针。 p开头

错误 err开头

日志

  1. 当你遇到问题可以通过debug功能定位出问题所在的时候

  2. if。。else。。或者switch时，进入分支的时候

  3. 核心代码流程性说明
